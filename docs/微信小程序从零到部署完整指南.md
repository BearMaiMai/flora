## 1. 前期准备

### 1.1 明确需求

在开始之前，你需要准备：

- **产品需求文档**：明确小程序要做什么功能
- **UI 设计稿**：页面布局和交互设计（推荐使用 Figma / Sketch / 即时设计）
- **后端接口**：如果需要服务端，提前规划 API 接口

### 1.2 所需资源

| 资源 | 说明 |
|------|------|
| 邮箱 | 未注册过微信公众平台的邮箱 |
| 身份证 | 个人主体需要实名认证 |
| 营业执照 | 企业主体需要（可选，个人也可注册） |
| 域名 + SSL 证书 | 后端服务器需要（必须 HTTPS） |
| 服务器 | 部署后端 API 使用（云开发可省略） |

### 1.3 个人 vs 企业主体对比

| 能力 | 个人主体 | 企业主体 |
|------|---------|---------|
| 微信支付 | ❌ 不支持 | ✅ 支持 |
| 获取用户手机号 | ❌ 不支持 | ✅ 支持 |
| 卡券功能 | ❌ 不支持 | ✅ 支持 |
| 客服消息 | ✅ 支持 | ✅ 支持 |
| 地理位置 | ✅ 支持 | ✅ 支持 |
| 认证 | ❌ 不支持 | ✅ 支持（300元/年）|

## 2. 注册小程序账号

### 2.1 注册步骤

1. 打开 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角 **「立即注册」**
3. 选择账号类型：**「小程序」**
4. 填写注册信息：
   - 邮箱（未在微信公众平台注册过的）
   - 密码
   - 验证码
5. 邮箱验证：打开邮箱点击激活链接
6. 选择主体类型：
   - **个人**：填写身份证信息 + 微信扫码验证
   - **企业**：填写营业执照信息 + 对公打款验证 或 法人微信扫码验证
7. 完成注册

### 2.2 获取 AppID

注册完成后：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入 **「开发」→「开发管理」→「开发设置」**
3. 记录你的 **AppID（小程序ID）** 和 **AppSecret（小程序密钥）**

> ⚠️ **重要**：AppSecret 只显示一次，务必安全保存！泄露后需要重置。

## 3. 完善小程序信息

### 3.1 基本信息设置

在微信公众平台 **「设置」→「基本设置」** 中填写：

- **小程序名称**：2~30 个字符，发布前必须设置（发布后修改需审核）
- **小程序头像**：清晰辨识，不超过 2M（发布后每月可修改 5 次）
- **小程序简介**：4~120 个字符
- **服务类目**：选择你的小程序所属的服务类目（**非常重要，影响审核**）

### 3.2 服务类目选择

常见类目示例：

| 类目 | 适用场景 | 是否需要资质 |
|------|---------|-------------|
| 工具 → 信息查询 | 工具类小程序 | 否 |
| 生活服务 → 生活服务 | 生活服务类 | 否 |
| 电商平台 → 电商平台 | 商城类 | 是（营业执照等） |
| 教育 → 在线教育 | 教育类 | 是 |
| 餐饮 → 餐饮服务 | 餐饮类 | 是 |

> ⚠️ 类目选错会导致审核不通过，请根据实际业务选择。

### 3.3 配置服务器域名

在 **「开发」→「开发管理」→「开发设置」→「服务器域名」** 中配置：

```
request 合法域名：   https://api.yourdomain.com
socket 合法域名：    wss://ws.yourdomain.com      （如需 WebSocket）
uploadFile 合法域名：https://upload.yourdomain.com  （如需上传文件）
downloadFile 合法域名：https://cdn.yourdomain.com   （如需下载文件）
```

**域名要求**：
- ✅ 必须是 **HTTPS** 协议（不支持 HTTP）
- ✅ 必须是 **已备案** 的域名（国内服务器）
- ❌ 不支持 IP 地址
- ❌ 不支持 localhost（仅开发时可在开发者工具中勾选"不校验"）

## 4. 开发环境搭建

### 4.1 下载微信开发者工具

1. 前往 [微信开发者工具下载页](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 选择对应操作系统版本（Windows / macOS / Linux）下载安装
3. 使用微信扫码登录

### 4.2 创建项目

1. 打开微信开发者工具
2. 点击 **「+」新建项目**
3. 填写信息：
   - **项目名称**：你的小程序名称
   - **目录**：选择本地开发目录
   - **AppID**：填入你的 AppID（测试可用测试号）
   - **开发模式**：小程序
   - **后端服务**：不使用云服务 / 微信云开发（按需选择）
   - **模板**：可选择基础模板或空白项目
4. 点击确定，项目创建完成

### 4.3 开发框架选择

| 框架 | 说明 | 适用场景 |
|------|------|---------|
| **原生开发** | 微信官方 WXML + WXSS + JS | 简单项目、追求性能 |
| **uni-app** | Vue 语法，跨平台 | 需同时兼容多端（H5、App等） |
| **Taro** | React/Vue 语法，跨平台 | React 技术栈团队 |
| **WePY** | 类 Vue 语法（渐不维护） | 不推荐新项目使用 |

> 💡 **建议**：初学者推荐原生开发，熟悉后可选择 uni-app 或 Taro 提高效率。

**index.js（页面逻辑）**
```javascript
const app = getApp()

Page({
  data: {
    list: [],
    loading: true
  },

  onLoad(options) {
    this.fetchData()
  },

  onPullDownRefresh() {
    this.fetchData()
    wx.stopPullDownRefresh()
  },

  fetchData() {
    this.setData({ loading: true })
    wx.request({
      url: 'https://api.yourdomain.com/list',
      method: 'GET',
      success: (res) => {
        this.setData({
          list: res.data.data,
          loading: false
        })
      },
      fail: (err) => {
        console.error('请求失败:', err)
        this.setData({ loading: false })
        wx.showToast({ title: '加载失败', icon: 'none' })
      }
    })
  },

  onItemTap(e) {
    const id = e.currentTarget.dataset.id
    wx.navigateTo({
      url: `/pages/detail/detail?id=${id}`
    })
  }
})
```

### 6.2 常用 API

```javascript
// ====== 路由导航 ======
wx.navigateTo({ url: '/pages/detail/detail?id=1' })   // 保留当前页，跳转

  fail(err) { console.error(err) }

// ====== 数据缓存 ======

  }
})

// ====== 用户信息（新版） ======
// 获取头像
<button open-type="chooseAvatar" bindchooseavatar="onChooseAvatar">获取头像</button>
// 获取昵称

      }
    })
  }

```

### 6.3 网络请求封装

```javascript
// utils/request.js
const BASE_URL = 'https://api.yourdomain.com'

const request = (options) => {
  return new Promise((resolve, reject) => {
    const token = wx.getStorageSync('token')
    wx.request({
      url: `${BASE_URL}${options.url}`,
      method: options.method || 'GET',
      data: options.data || {},
      header: {
        'content-type': 'application/json',
        'Authorization': token ? `Bearer ${token}` : '',
        ...options.header
      },
      success: (res) => {
        if (res.statusCode === 200) {
          resolve(res.data)
        } else if (res.statusCode === 401) {
          // token 过期，重新登录
          wx.removeStorageSync('token')
          wx.redirectTo({ url: '/pages/login/login' })
          reject(res)
        } else {
          wx.showToast({ title: res.data.message || '请求失败', icon: 'none' })
          reject(res)
        }
      },
      fail: (err) => {
        wx.showToast({ title: '网络异常', icon: 'none' })
        reject(err)
      }
    })
  })
}

module.exports = { request }
```

### 6.4 分包加载（优化包体积）

当小程序代码包超过 **2MB** 时，需要使用分包：

```json
// app.json
{
  "pages": [
    "pages/index/index",
    "pages/mine/mine"
  ],
  "subpackages": [
    {
      "root": "packageA",
      "name": "packageA",
      "pages": [
        "pages/detail/detail",
        "pages/list/list"
      ]
    },
    {
      "root": "packageB",
      "name": "packageB",
      "pages": [
        "pages/setting/setting"
      ]
    }
  ],
  "preloadRule": {
    "pages/index/index": {
      "network": "all",
      "packages": ["packageA"]
    }
  }
}
```

**包体积限制**：
- 单个分包 / 主包：**不超过 2MB**
- 所有包总大小：**不超过 20MB**（开通分包后）

## 8. 云开发（可选）

微信云开发提供 **免服务器** 的后端能力，适合快速开发。

### 8.1 开通云开发

1. 在开发者工具中点击 **「云开发」** 按钮
2. 开通云开发环境（免费额度可满足小型项目）
3. 创建环境（建议创建两个：开发环境 + 生产环境）

### 8.2 云开发能力

| 能力 | 说明 |
|------|------|
| **云数据库** | NoSQL 数据库（类 MongoDB） |
| **云存储** | 文件存储（图片、文件等） |
| **云函数** | 运行在云端的 Node.js 函数 |
| **云调用** | 云函数中直接调用微信 API |
| **HTTP API** | 提供 HTTP 接口供外部系统调用 |

### 8.3 云函数示例

```javascript
// cloudfunctions/getList/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  try {
    const { page = 1, pageSize = 10 } = event
    const skip = (page - 1) * pageSize

    const { data } = await db.collection('articles')
      .orderBy('createTime', 'desc')
      .skip(skip)
      .limit(pageSize)
      .get()

    const { total } = await db.collection('articles').count()

    return { code: 0, data, total }
  } catch (err) {
    return { code: -1, message: err.message }
  }
}
```

```javascript
// 小程序端调用云函数
wx.cloud.callFunction({
  name: 'getList',
  data: { page: 1, pageSize: 10 },
  success(res) {
    console.log('数据列表:', res.result.data)
  }
})
```

## 9. 体验版测试

1. 在微信开发者工具中，点击右上角 **「上传」** 按钮
2. 填写：
   - **版本号**：如 `1.0.0`（遵循语义化版本规范）

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入 **「管理」→「版本管理」**
3. 在 **「开发版本」** 列表中找到刚上传的版本
4. 点击 **「设为体验版」**

1. 进入 **「管理」→「成员管理」**
2. 在 **「体验成员」** 中添加微信号
3. 个人主体最多 **15** 个体验成员，企业主体最多 **50** 个

### 9.4 体验版测试要点

- [ ] 所有页面是否正常加载
- [ ] 网络请求是否正常（注意域名配置）
- [ ] 登录流程是否顺畅
- [ ] 各种机型适配（iPhone / Android / 不同屏幕尺寸）
- [ ] 弱网环境下的表现
- [ ] 异常情况处理（无数据、网络错误等）
- [ ] 分享功能是否正常
- [ ] 返回导航是否正确

## 10. 提交审核

### 10.1 审核前检查

> ⚠️ 以下内容会直接影响审核结果，务必仔细检查！

**必查项**：
- [ ] 小程序名称、头像、简介已设置
- [ ] 服务类目选择正确
- [ ] 服务器域名已配置且可访问
- [ ] 所有页面内容与所选类目匹配
- [ ] 无测试数据、无 "lorem ipsum" 占位文本
- [ ] 无空白页面、无死链
- [ ] 隐私协议已配置（**2023年9月起强制要求**）

**隐私协议配置**：
1. 在微信公众平台 **「设置」→「基本设置」→「服务内容声明」** 中填写隐私协议
2. 在代码中使用隐私弹窗组件：

```json
// app.json
{
  "__usePrivacyCheck__": true
}
```

```xml
<!-- 隐私弹窗组件 -->
<view wx:if="{{showPrivacy}}" class="privacy-modal">
  <view class="privacy-content">
    <text>在你使用本小程序之前，请仔细阅读</text>
    <text class="link" bindtap="openPrivacy">《隐私协议》</text>
    <view class="privacy-btns">
      <button bindtap="rejectPrivacy">拒绝</button>
      <button id="agree-btn" open-type="agreePrivacyAuthorization"
              bindagreeprivacyauthorization="agreePrivacy">同意</button>
    </view>
  </view>
</view>
```

### 10.2 提交审核步骤

1. 登录微信公众平台
2. 进入 **「管理」→「版本管理」**
3. 在 **「开发版本」** 中点击 **「提交审核」**
4. 填写审核信息：
   - 配置功能页面（设置主要入口页面的标题和标签）
   - 是否使用插件
   - 是否为小游戏
5. 提交

### 10.3 审核时间

- 普通审核：**1~7 个工作日**（通常 1~3 天）
- 加急审核：每年有 **3 次** 加急机会（约几小时内完成）
- 审核高峰期（如节假日前）可能更慢

### 10.4 常见审核驳回原因

| 驳回原因 | 解决方案 |
|---------|---------|
| 服务类目不匹配 | 修改类目或调整小程序内容 |
| 小程序功能不完整 | 完善所有页面功能，去除空白页 |
| 含有测试数据 | 替换为正式数据 |
| 隐私协议缺失 | 添加隐私协议弹窗 |
| 诱导分享/关注 | 去除强制分享、关注的逻辑 |
| 存在虚假信息 | 确保内容真实合规 |
| 包含广告组件未报备 | 使用微信官方广告组件或去除广告 |
| 需要资质但未提交 | 上传相关资质证明 |

## 11. 正式发布上线

### 11.1 发布流程

1. 审核通过后，登录微信公众平台
2. 进入 **「管理」→「版本管理」**
3. 在 **「审核版本」** 中点击 **「发布」**
4. 确认发布

> 💡 发布后一般 **5~10 分钟** 内生效，用户即可搜索和使用。

### 11.2 全量发布 vs 灰度发布

**灰度发布**（分阶段发布）：
1. 点击发布时选择 **「分阶段发布」**
2. 设定比例（如先发布给 10% 的用户）
3. 观察线上表现
4. 逐步提高比例直至全量

**建议**：首次发布可直接全量，后续重大更新使用灰度发布。

### 11.3 发布后验证

- [ ] 搜索小程序名称能正常找到
- [ ] 扫描小程序码能正常打开
- [ ] 核心功能正常使用
- [ ] 分享卡片显示正常

### 12.1 数据分析

微信公众平台 **「统计」** 页面提供：
- **概况**：访问人数、新增用户、活跃用户
- **访问分析**：PV / UV、访问时长、页面分析
- **实时统计**：当天数据
- **自定义分析**：事件上报和漏斗分析

### 12.2 自定义数据上报

```javascript
// 上报自定义事件
wx.reportAnalytics('purchase_event', {
  product_id: '12345',
  product_name: '商品名称',
  price: 99.9
})

// 上报监控数据
wx.reportMonitor('api_error_count', 1)
```

### 12.3 错误监控

```javascript
// app.js 中捕获全局错误
App({
  onError(msg) {
    // 上报错误信息到自己的监控平台
    wx.request({
      url: 'https://api.yourdomain.com/report/error',
      method: 'POST',
      data: {
        message: msg,
        time: new Date().toISOString(),
        systemInfo: wx.getSystemInfoSync()
      }
    })
  },

  onPageNotFound(res) {
    wx.redirectTo({ url: '/pages/index/index' })
  }
})
```

### 12.4 版本更新提示

```javascript
// app.js
App({
  onLaunch() {
    this.checkUpdate()
  },

  checkUpdate() {
    if (!wx.canIUse('getUpdateManager')) return

    const updateManager = wx.getUpdateManager()

    updateManager.onCheckForUpdate((res) => {
      if (res.hasUpdate) {
        console.log('发现新版本')
      }
    })

    updateManager.onUpdateReady(() => {
      wx.showModal({
        title: '更新提示',
        content: '新版本已准备好，是否重启应用？',
        success(res) {
          if (res.confirm) {
            updateManager.applyUpdate()
          }
        }
      })
    })

    updateManager.onUpdateFailed(() => {
      wx.showModal({
        title: '更新提示',
        content: '新版本下载失败，请检查网络后重试',
        showCancel: false
      })
    })
  }
})
```

## 13. 常见问题 FAQ

### Q1：开发时请求报错 "不在以下合法域名列表中"

**解决**：
1. 开发阶段：在开发者工具 **「详情」→「本地设置」** 中勾选 **「不校验合法域名」**
2. 正式环境：在微信公众平台配置服务器域名

### Q2：小程序包体积超过 2MB

**解决**：
1. 使用 **分包加载**（参考第 6.4 节）
2. 图片资源放到 CDN，不要放在小程序包内
3. 移除未使用的代码和资源
4. 使用开发者工具的 **「代码依赖分析」** 查看包大小分布

### Q3：审核不通过怎么办？

**解决**：
1. 仔细阅读驳回原因说明
2. 根据原因修改代码或配置
3. 重新上传并提交审核
4. 如有争议可申诉

### Q4：如何让别人搜索到我的小程序？

**优化搜索排名**：
1. 完善小程序名称和简介（包含关键词）
2. 配置 `sitemap.json` 允许微信爬虫索引
3. 保持活跃用户量和使用频次
4. 添加「页面收录」配置

### Q5：后端服务器如何选择？

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 微信云开发 | 免运维、免域名备案、快速上手 | 灵活性有限、供应商锁定 | 个人/小型项目 |
| 腾讯云/阿里云轻量服务器 | 灵活、便宜 | 需自行运维 | 中小型项目 |
| 腾讯云/阿里云 ECS | 高性能、可扩展 | 成本较高、运维复杂 | 中大型项目 |
| Serverless（云函数） | 按量付费、免运维 | 冷启动延迟 | 访问量不稳定的项目 |

### Q6：开发到上线需要多长时间？

| 项目规模 | 预估时间 |
|---------|---------|
| 简单展示类（3~5 个页面） | 1~2 周 |
| 工具类（核心功能明确） | 2~4 周 |
| 电商/社交类（功能复杂） | 1~3 个月 |
| 大型平台类 | 3~6 个月+ |
